New tag indexing features with more shell scripts
<2026-05-07 Thu> :test:meta:tech:

Hello again, just the morning after. I've figured out a few things, namely:

  • Refurbishing the site CSS
  • Basic tagging features!!

As you can probably tell in regards to the first item, I am not talented with web design, only making humble blogs like I know I could make. At least it's better than everything being arranged in a full width series of blocks, and a navigation bar at the top. I got inspiration from MediaWiki's look, since I use that a lot. I figured side columns are an acceptable web design tradition even now, hopefully.

And in regards to tagging features, YES! There are tags for blog posts, so you don't have to sift through all the titles to find what's relevant. I've done multiple things actually. Posts always were tagged, but I now have added an automatic index for every tag present, that appears on the sidebar, much like Wordpress or Blogspot. I'll probably work on some feature like automatic linking to the index every time the tag is added, and numbering articles per tag in the sidebar. But for now, this is phenomenal organisation, no?

How does it work? Well, I've tied together some more shell scripts into my export.sh file that

  1. Searches in the blog directory for pages matching tags
  2. Grabs its title and filename
  3. Constructs a list of links to matching files
  4. Pipes all that with some formatting into the index for a tag
  5. Makes a list of tags as linked to their indices
#!/bin/bash

mkdir -p tag

grep '#+FILETAGS: ' $(find blog/*.org) \
| grep -oEe ' :.*$' \
| tr -d ' \n' \
| sed 's;:\{2,\};:;' \
| sed 's;^:;;' \
| tr ':' '\n' \
| sort | uniq > tags

export_tag () {
    echo '#+TITLE: ' $1 '(tag)' > tag/$1.org
    grep -HEe '^#\+TITLE:' \
    $(grep -HEe "^#\+(FILETAGS:.*:($1):)" $(find blog/*.org) \
      | sed 's;\(^.*\):#+FILETAGS.*$;\1;') \
    | sed 's;\(^.*\):#+TITLE: *\(.*$\); - [[../\1][\2]];' >> tag/$1.org
}

for x in $(cat tags)
do
  export_tag $x;
done


echo '#+BEGIN_EXPORT html' > tags-index.org
cat tags \
| sed 's;\(^.*$\);<a href=\"/tag/\1.html\">\1</a>;' \
| tr '\n' '|' \
| sed 's;|$;\n;' >> tags-index.org
echo '#+END_EXPORT' >> tags-index.org

Some smart tricks I've done here are to embed this as HTML within a .org file, so I may include it in the header.org, which gets exported to HTML complete, and is kept on every webpage. The illusion of a dynamic website, when it's really a very well maintained static site.

I don't use many special tools in my script, as it's just extraction and text substitution that come into play. I have a good enough knowledge of regex to enable this, too.

It's really amazing that such a simple program can do so much with the construction of even simpler tools together. This is an example of the kind of stuff that shell scripts come in handy for: automation. Automating all the boring algorithmic tasks.

Complexity is inversely proportional to convolution. The more complex something is, the less convoluted it is, et vice versa.

I don't know if I've explained this before, but I've had my fair share of tedious manual writing of HTML. Before my C++ knowledge would've been able to solve the problem, before I was even on a system that offered , I made webpages with HTML and CSS. It was tiring to proofread everything. But now, after studying the Unix shell, I can finally apply my knowledge, and understand the magic therein. Don't you think we were meant to make our computers work for us, approach automating everything?
